前几天，在与表姐聊天过程中聊到了云服务的问题时，我向她展示了自己刚搭建好的个人博客网站。作为我的博客第一个访客的福利，我答应给她写一篇零代码基础的教程。

这篇教程文除了详细的搭建博客的步骤以外，还会包含各种细节问题的填坑以及发文的详细流程。但本文也仅限于总结我在搭建博客时的具体方案，涉及其他不同的个性化方案，会稍有提及，但并不会一一详解。

概述

首先，我们先弄清楚Hexo和GitHub是什么?

通过Hexo官网的介绍，我们可以了解到Hexo是一个一个快速、简洁且高效的博客框架，它基于Node.js，使用Markdown来解析文章。

而GitHub是一个面向开源和私有软件项目的托管平台,也是一个开源代码库以及版本控制系统。

而GitHub有一个GitHubPages的功能，用于设置你的主页或是你的项目主页。由此我们可以利用这个功能，根据Markdown语法写好我们的博文，使用Hexo上传至GitHub中，然后别人访问我们的主页时，就可以看到Hexo通过靓丽的主题生成的静态网页了。在搭建好我们的博客以后，我们只需使用Markdown语法写好博文上传至GitHub就可以了，而无需关注网页具体生成细节，是不是很简单呢？

安装Node.js

在安装Hexo之前我们首先要安装Node.js和Git。

下载Node.js（这里给出的是windows系统的64位安装包，请根据自己的系统选择合适的安装包。另该版本使用chrome下载后会有危险警示，无视它好了。）安装过程全部默认设置就可以了，安装路径可自行修改。

安装之后，按Win+R打开”运行程序”,输入cmd，打开“命令提示符窗口”，输入node -v，回车，出现版本号，则表示安装成功。

安装Git

下载Git(进入网页后，稍等一下会自动弹出下载框，安装包适用于Windows系统64位)。打开安装包，一路next下去就可以了，有需要的可自行更改安装路径。

安装之后，按Win+R打开”运行程序”,输入cmd，打开“命令提示符窗口”，输入git version，回车，出现版本号，则表示安装成功。

安装Hexo

我们可以利用git来安装Hexo。

首先，我们要新建一个文件夹放置我们的博客文件(我的是在“D:\linftp”)，打开该文件夹,点击鼠标右键，选择Git Bash Here。

输入npm config set registry https://registry.npm.taobao.org(这一步意义在于把npm默认安装源地址改为淘宝的镜像站地址，原因是因为默认安装源地址速度实在太慢，还可能因为“被墙”问题导致访问不成功。)，回车。

输入npm install hexo-cli –g，回车，稍等一会即可完成，可能会有“waring”，无视即可。

输入hexo -v，回车，出现下图表示安装成功。

输入hexo init，回车，Hexo会在该文件夹下初始化生成博客所需的所有文件。（可能会有几个warning，无视即可。）

输入npm install，回车，安装依赖包。（可能会有几个warning，无视即可。）

至此，Hexo已经安装完毕了，输入hexo g生成静态网页，输入hexo s启动本地服务器上。这时，打开浏览器，输入http://localhost:4000/即可访问到我们刚建好的博客了。

在Git Bash界面按Ctrl+C即可关闭本地服务器。

现象：访问GitHub官网，发现网页加载不出来。按F12查看浏览器console窗口可知在访问“assets-cdn.github.com”与“avatars2.githubusercontent.com”时出现连接超时现象，可能“被墙”(如果你的是连接其他网页地址出现超时，也可使用下面的方法。请根据你的实际情况操作。)。

解决方案：
1、登录DNS查询工具，输入“assets-cdn.github.com”查询。

3、打开“C:\Windows\System32\drivers\etc”目录下的hosts文件，可以使用记事本打开。在最下面按照下图的格式填上你刚才记录的IP地址。

4、按照同样的方式查询“avatars2.githubusercontent.com”，同样在最下面加上你查询到的IP地址。最终在“hosts”文件下添加了6条DNS解析地址。

5、保存“hosts”文件，按WIN+R，输入cmd,打开“命令提示符窗口”，输入ipconfig /flushdns，回车，刷新DNS缓存。然后重新访问Git官网，此时访问正常。

如何添加一篇文章

这时候我们要做的是如何部署到GitHub上，回到博客根目录文件夹。鼠标右键，点击“Git Bash Here”，然后输入hexo c,回车,输入hexo g ，回车，输入hexo d，回车，即可完成部署。这个时候，打开你的浏览器，输入域名，（我的是linftp.com），即可看到你刚才写好的文章。如果看不到，那么证明你前面某个步骤出错了。

现在我们来看一下这三条命令代表什么意思。先附上Hexo官网文档。

hexo c全称hexo clean，它的作用是清除缓存的文件以及已生成的静态文件。

hexo g全称hexo generate,它的作用是重新生成静态文件。

hexo d全称hexo deploy,它的作用是把生成的静态文件部署到GitHub上。

这三条命令都需要在博客根目录文件夹下，鼠标右键，点击“Git Bash Here”，然后再输入。

值得一提的是，你在输入hexo c和hexo g后，可以先不急着部署到GitHub上，而是先在本地服务器上浏览一下效果。具体做法是输入hexo s，然后打开浏览器，输入 http://localhost:4000/查看效果，查看后，回到“Git Bash”界面，按Ctrl+C退出本地服务器，输入hexo d部署到GitHub上。

我们重新梳理一下新建文章的顺序，在博客根目录文件夹下，鼠标右键，点击“Git Bash Here”，输入hexo new post "file name"，自己取好博文文件名，然后写好.md文件，放到博客根目录的source->_posts下。回到博客根目录文件夹。鼠标右键，点击“Git Bash Here”，然后输入hexo c,回车,输入hexo g ，回车，输入hexo d，回车，即可完成部署。

尽管主题已经让我们的博客变得各式各样，多姿多彩，我们依旧可以通过安装插件以及更改配置文件的方式使我们的博客更富个性化。
